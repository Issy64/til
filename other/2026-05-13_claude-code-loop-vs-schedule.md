## 💡 学んだこと（要約）
### 1. 何を知ったか（一言）
時間駆動の繰り返し実行には `/loop`（ローカル）か `/schedule`（クラウド routine）を使う。`hooks` はイベント駆動なので「30分おき」のような時間駆動には使えない。

### 2. 以前の知識とどう繋がったか
hooks（`settings.json` の `PostToolUse` / `Stop` など）は「特定のツール呼び出しの前後」に発火するイベント駆動の仕組みだと知っていたが、時間駆動との区別を明示的に整理できていなかった。

## 使い分けまとめ

| 方式 | 実行場所 | 特徴 |
|------|---------|------|
| `/loop [間隔] /スキル名` | ローカルセッション | Claude Code を起動している間だけ動く。認証・権限はローカルのまま使える |
| `/schedule`（routine） | クラウド（claude.ai） | cron で常駐実行。ローカルを閉じていても動く |
| hooks（settings.json） | ローカル | イベント駆動（ツール前後・Stop時など）。時間駆動には対応しない |

## routine 利用時の注意点

- 承認プロンプトは出ない。事前許可は routine 作成時の Connectors / Permissions タブで設定する
- `Status: green` はインフラが正常という意味であり、タスクが成功したことではない。中身を個別確認する
- `gh pr create` のような書き込み系コマンドは「変更があるときだけ実行、なければスキップ」のように条件付き＆自己完結で書く
- ローカルの認証や権限ルールをそのまま使いたい場合は routine ではなく `/loop` が素直
